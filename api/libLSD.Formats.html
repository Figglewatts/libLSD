Namespace libLSD.Formats

Classes

TMDPrimitivePacket

A TMD primitive packet. For the most part, this is the raw data that gets sent to the GPU to draw the object. There's a bunch of different types of these - quads and tries, all with different properties like shading, texture mapping, and lighting.

Structs

LBD

An LBD is a single chunk of a level in LSD. Levels consist of a bunch of chunks arranged in a certain way. The chunks can either be arranged horizontally or vertically. If they are arranged vertically then they are all just put on the same location. Each chunk tile uses its TileHeight to offset above. If they are arranged horizontally, it is in a honeycomb-like pattern.

An LBD is used to store a bunch of things:

  • Chunk tiles in a TMD file
  • Chunk tile layout
  • Chunk interactive objects in an MML file (this contains MOM files).

LBDHeader

The header of an LBD file, containing various pieces of metadata about the file.

LBDTile

A single tile in an LBD chunk. References a TMD object in the tiles TMD and has a bunch of metadata describing rotation, height, and extra tiles.

MML

An MML file is a container for MOM interactive object files, found in LBD files.

MOM

A MOM file is an interactive object in LSD. It has a TMD, and an MOS file, which is a container for animation data.

MOS

An MOS file is a container for TOD animation data. It's found in MOM files.

TIM

A TIM file is a PSX texture image. It essentially stores colour data, supporting a variety of formats.

TIMColorLookup

A TIM colour lookup table, used to store colours in an efficient manner.

TIMHeader

The header of a TIM file, containing its ID and flags

TIMPixelData

Stores the actual pixel (color) data of the TIM file.

TIX

A TIX file is a collection of TIM texture data, organised into VRAM pages. Each level has TIX files containing every texture used in the level, so they can all be loaded into PSX VRAM at once.

A TIX file is essentially a collection of 'chunks'

TIXChunk

A TIX chunk, containing a number of TIM textures.

TIXHeader

The header of a TIX file.

TMD

A TMD file stores 3D model data. They contain vertex data, normal data, and primitive data. The primitive data is different 'packets' that get sent to the GPU to tell it to draw triangles/quads with different lighting, shading and texturing.

TMDs can contain multiple 'objects'. These are essentially independent models. In LSD they are used to store separate tiles in an LBD file, or separate parts of animated models (like arms/legs etc).

TMDColorLookup

Indicates the position in VRAM where a color lookup table is stored.

TMDHeader

The header of a TMD file.

TMDNormal

A normal, as stored in a TMD.

TMDObject

An object within the TMD. Stored in the TMD's object table. Contains pointers to vertices, normals, and primitives.

TMDTexture

Contains information on a texture/sprite on a TMD primitive packet.

TOD

The TOD format is used for storing information along the flow of time, relative to a 3-dimensional object. For each frame in a 3D animation, the TOD file describes the required data to pose a 3D model for that frame.

TODFrame

A single frame in a TOD animation. A frame consists of a bunch of packets that describe what to do with different components of a 3D model (like transformations etc).

TODHeader

The header of a TOD file.

TODPacket

A TOD packet. A packet describes how to do one specific action in a frame, like a transformation.
